## Threat Model Seeds

| Threat Ref | Threat | Required Mitigation |
|------------|--------|---------------------|
| T-12.1-01 | Data leakage during calibration makes confidence look better than reality | Fit calibrators only on validation/calibration slices and test with future-row mutation checks |
| T-12.1-02 | Tampered threshold artifact enables unsafe trades | Validate schema, numeric ranges, class labels, and artifact path under saved model directory |
| T-12.1-03 | Missing gate reason hides why live trades were blocked or allowed | Persist structured DecisionAudit for pass/block/weaken/veto outcomes |
| T-12.1-04 | Bad challenger gets promoted because only one metric improves | Require profit factor not worse, drawdown not worse, calibration not worse, and minimum trade count |
| T-12.1-05 | Retraining triggers fire on noisy raw confidence | Trigger on calibrated degradation metrics and sustained evidence windows |
| T-12.1-06 | Phase 12.3 specialist bypasses core ensemble | DecisionGovernor must reject specialist-only trade creation |
