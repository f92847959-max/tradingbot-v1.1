# Plan 04-02: Dynamic ATR-Based Labels and Backtester Alignment

<prerequisite>
Plan 04-01 complete: `strategy/regime_detector.py` and `strategy/regime_params.py` exist
with `MarketRegime`, `RegimeDetector`, and `REGIME_PARAMS`.

The `LabelGenerator` in `ai_engine/training/label_generator.py` currently uses fixed
`tp_pips=50` and `sl_pips=30` for label generation. The `Backtester` in
`ai_engine/training/backtester.py` uses the same fixed pips for trade evaluation.
Meanwhile, live execution (`entry_calculator.py`, `ensemble.py`) already uses ATR-based
SL/TP. This creates a training-execution mismatch.

This plan aligns label generation and backtesting with live execution by making them
ATR-based.
</prerequisite>

<objective>
Modify `LabelGenerator` to support ATR-based dynamic TP/SL distances per candle (using
the `atr_14` column already present in the DataFrame). Modify `Backtester.run_simple()`
to evaluate trades with per-trade ATR-based TP/SL instead of fixed pips. Update
`ModelTrainer` and `TrainingPipeline` to pass ATR mode configuration. Update
`train_models.py` CLI to support ATR-based mode.

This eliminates the training-execution mismatch: labels, backtesting, and live trading
all use ATR-based TP/SL.

Requirement coverage: STRAT-01 (dynamic TP/SL based on ATR instead of fixed pips).
</objective>

<specification>

### Changes to `generate_labels`:

When `self.use_dynamic_atr is True`:
1. Read `df["atr_14"]` column. If missing, raise `ValueError("atr_14 column required for dynamic ATR mode")`.
2. Handle NaN ATR values: `median_atr = np.nanmedian(atr_values)`, then `atr = np.where(np.isnan(atr), median_atr, atr)`.
3. If `median_atr` is also NaN (all NaN), fall back to fixed pips and log a warning.
4. Compute per-candle distances:
   ```python
   tp_dist = atr * self.tp_atr_multiplier
   sl_dist = atr * self.sl_atr_multiplier
   ```
5. Apply minimum floors:
   ```python
   tp_dist = np.maximum(tp_dist, self.min_tp_pips * self.pip_size)
   sl_dist = np.maximum(sl_dist, self.min_sl_pips * self.pip_size)
   ```
6. Apply costs (same logic as current, but per-candle arrays):
   ```python
   buy_tp_dist = tp_dist + self.total_cost    # TP further
   buy_sl_dist = sl_dist - self.total_cost    # SL closer
   buy_sl_dist = np.maximum(buy_sl_dist, self.pip_size)  # min 1 pip
   sell_tp_dist = tp_dist + self.total_cost
   sell_sl_dist = sl_dist - self.total_cost
   sell_sl_dist = np.maximum(sell_sl_dist, self.pip_size)
   ```
7. Call a new method `_vectorized_labeling_dynamic()` that accepts per-candle arrays
   instead of scalar distances.

When `self.use_dynamic_atr is False`: unchanged behavior (current code path).

<notes>
- CRITICAL: The `_vectorized_labeling_dynamic` method is nearly identical to
  `_vectorized_labeling` but indexes into per-candle arrays. Do NOT try to
  consolidate them into one method -- the scalar vs array paths have different
  performance characteristics and the code is clearer kept separate.

- The `use_dynamic_atr=True` default in `ModelTrainer` means new training runs
  automatically use ATR-based labels. The `--no-dynamic-atr` CLI flag provides
  backward compatibility.

- ATR NaN handling is critical: ATR-14 produces NaN for the first ~14 candles.
  The label generator runs BEFORE warmup removal (pipeline Step 3 vs Step 4).
  Use `np.nanmedian` of the non-NaN portion to fill NaN ATR values.

- The minimum TP/SL floor (`min_tp_pips=5.0`, `min_sl_pips=3.0`) prevents
  degenerate labels in very low volatility periods. For Gold at pip_size=0.01,
  this means minimum TP=$0.05 and SL=$0.03.

- The `Backtester.run_simple` ATR enhancement is important for walk-forward
  evaluation accuracy but is NOT yet called with ATR values in the pipeline.
  That wiring will happen when Phase 5 backtesting is built. For now, the
  capability exists and is tested.

- Existing tests that create `LabelGenerator()` without arguments will still
  work because `use_dynamic_atr` defaults to `False` in `LabelGenerator` itself.
  Only `ModelTrainer` defaults to `True`.
</notes>
