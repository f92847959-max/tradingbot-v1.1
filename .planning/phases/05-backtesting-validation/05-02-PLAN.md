<task type="auto">
  <name>Task 1: Create CLI backtest script</name>
  <files>scripts/run_backtest.py</files>
  <action>
    Create `scripts/run_backtest.py` following the same CLI pattern as `scripts/train_models.py`.

    Arguments:
    - `--version-dir` (required): Path to the trained model version directory (e.g., `ai_engine/saved_models/v001_20260308_143022`)
    - `--csv` (optional): Path to CSV with OHLCV data for backtesting. If not provided, use `--broker` or `--synthetic`.
    - `--broker` (optional): Fetch data from Capital.com (reuse `fetch_broker_data` from train_models.py or duplicate the pattern)
    - `--synthetic` (optional): Number of synthetic candles to generate for testing
    - `--count` (optional, default 1000): Number of candles to fetch from broker
    - `--timeframe` (optional, default "5m"): Candle timeframe
    - `--commission` (optional, default 0.0): Commission per trade in pips
    - `--output` (optional): Path to save the JSON backtest report. Defaults to `{version_dir}/backtest_report.json`

    Flow:
    1. Parse args, set up logging
    2. Load data (from CSV, broker, or synthetic) -- reuse `generate_synthetic_data` by importing from train_models or duplicating
    3. Compute features using `FeatureEngineer().create_features(df, timeframe)` to get feature columns
    4. Generate labels using `LabelGenerator` with params from version.json (needed for y_true in backtest)
    5. Remove warmup period (200 candles) to match training pipeline
    6. Separate X, y arrays (same as pipeline steps 4-5)
    7. Extract ATR values from the DataFrame if dynamic ATR mode (from `atr_14` column after feature computation)
    8. Create `BacktestRunner(version_dir=args.version_dir, commission_per_trade_pips=args.commission)`
    9. Call `runner.run(X, y, feature_names, atr_values=atr_values)`
    10. Print report via `print_backtest_report(result["report"], result["consistency"])`
    11. Save JSON report to output path
    12. Print summary: consistency PASS/FAIL, key aggregate metrics

    Important: The script must add the project root to sys.path (same pattern as train_models.py). Import FeatureEngineer and LabelGenerator to compute features and labels on the data, because the backtest runner expects pre-computed X/y arrays.

    Handle the case where `--version-dir` does not exist or does not contain version.json with a clear error message.
  </action>
  <verify>
    <automated>python scripts/run_backtest.py --help 2>&1 | head -20</automated>
  </verify>
  <done>CLI script accepts version-dir and data source args, runs backtest, prints report with consistency check, saves JSON. `--help` shows all options.</done>
</task>
