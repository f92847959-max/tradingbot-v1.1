  <action>
    **Create `ai_engine/training/backtest_runner.py`:**

    ```python
    class BacktestRunner:
        """Standalone out-of-sample backtest orchestrator.

        Loads a trained model version (model pkl + scaler pkl + version.json)
        and runs walk-forward backtesting on the OOS test windows with full
        cost modeling (spread + slippage + commission).
        """
    ```

    Constructor accepts:
    - `version_dir: str` -- path to the version directory
    - `commission_per_trade_pips: float = 0.0` -- configurable commission

    On init, load:
    1. `version.json` from version_dir -> extract `feature_names`, `label_params` (use_dynamic_atr, tp_atr_multiplier, sl_atr_multiplier, tp_pips, sl_pips, spread_pips), trade filter params (xgboost_trade_min_confidence, xgboost_trade_min_margin), and `walk_forward.windows` (the stored window boundaries).
    2. XGBoostModel from `xgboost_gold.pkl` via `.load()`
    3. LightGBMModel from `lightgbm_gold.pkl` via `.load()`
    4. FeatureScaler from `feature_scaler.pkl` via `.load()`

    Method `run(X: np.ndarray, y: np.ndarray, feature_names: List[str], atr_values: Optional[np.ndarray] = None) -> Dict[str, Any]`:
    - Use the window boundaries stored in version.json (from `walk_forward.windows`). Extract test_start/test_end from each window entry. If version.json does not have explicit indices (older format), fall back to `calculate_walk_forward_windows()`.
    - For each window's TEST portion:
      1. Slice X_test, y_test (and atr_test if atr_values provided) using window boundaries
      2. Create a DataFrame with feature_names columns from X_test, then transform with loaded scaler
      3. Predict with loaded XGBoost model -> probabilities
      4. Apply trade filter: `probs_to_trade_signals(y_probs, min_confidence, min_margin)`
      5. Create a `Backtester(initial_balance=10000, tp_pips=..., sl_pips=..., spread_pips=..., slippage_pips=0.5, commission_per_trade_pips=self.commission)` per research recommendation (each window independent, fresh 10k balance)
      6. Run `backtester.run_simple(predictions=signals, actual_labels=y_test_slice, atr_values=atr_test_slice, tp_atr_multiplier=..., sl_atr_multiplier=...)` when dynamic ATR mode, else without atr_values
      7. Collect the report dict as per-window result, adding window_id and window boundaries
    - After all windows: call `generate_backtest_report()` and `check_consistency()` from backtest_report module
    - Return dict with: `per_window_results`, `report`, `consistency`

    **IMPORTANT decisions per STATE.md and research:**
    - Each window is independent with fresh `initial_balance=10000` (research recommendation)
    - Aggregate PF from total gross_profit / total gross_loss (not averaged) per STATE.md decision
    - Zero-trade windows excluded from 60% positive calculation but flagged
    - Dynamic ATR: read `label_params.use_dynamic_atr` from version.json. If True, atr_values MUST be provided to `run()` and passed to `run_simple()`
    - Use XGBoost as the primary model for OOS evaluation (it's the one that gets SHAP-pruned and is the "best model" candidate)

    **Create `ai_engine/training/backtest_report.py`:**

    Functions:
    1. `generate_backtest_report(per_window_results: List[Dict], version_info: Dict) -> Dict`:
       - Build per-window entries: window_id, n_trades, total_pips, win_rate, profit_factor, sharpe_ratio, max_drawdown_pct, grade
       - Compute aggregate metrics from combined trades (total gross_profit / total gross_loss for PF, combined pips array for Sharpe with sqrt(2600))
       - Include version metadata, report_date, summary
       - Return JSON-serializable dict

    2. `check_consistency(per_window_results: List[Dict]) -> Dict`:
       - Count windows with trades where total_pips > 0 -> positive_windows
       - positive_pct = positive_windows / windows_with_trades (excluding zero-trade windows)
       - dd_violations = windows where max_drawdown_pct > 20.0
       - Return: n_windows, windows_with_trades, zero_trade_windows, positive_windows, positive_pct, passes_60pct, dd_violations count, passes_20pct_dd, overall_pass
       - overall_pass = passes_60pct AND passes_20pct_dd

    3. `print_backtest_report(report: Dict, consistency: Dict) -> None`:
       - Log formatted report to logger (similar to `print_training_report()` in walk_forward.py)
       - Show per-window table with key metrics
       - Show aggregate metrics
       - Show consistency check results (PASS/FAIL)

    Add all tests to `tests/test_backtest_runner.py`. For testing, create a helper that sets up a mock version directory with a simple model, scaler, and version.json. Use synthetic data (similar to the pattern in train_models.py's `generate_synthetic_data`). Since loading real models requires training, the tests should:
    - Test `check_consistency()` and `generate_backtest_report()` with hand-crafted per-window result dicts (unit tests, no model needed)
    - Test `BacktestRunner.__init__()` with a mock version directory (create temp dir with version.json + model pkls using synthetic trained models from conftest or similar)
    - If model training in tests is too heavy, test the report/consistency functions thoroughly and test the runner's logic with mocked model predictions
