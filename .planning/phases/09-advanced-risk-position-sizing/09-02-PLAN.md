    Class MonteCarloSimulator:

    __init__(self, ruin_threshold: float = 0.5):
        self.ruin_threshold = ruin_threshold  # 50% drawdown = "ruin"

    simulate(
        self,
        win_rate: float,
        avg_win: float,
        avg_loss: float,
        num_trades: int = 200,
        num_paths: int = 1000,
        initial_equity: float = 10000.0,
        position_fraction: float = 0.02,  # risk 2% per trade
        seed: int | None = None,
    ) -> SimulationResult:
        Algorithm:
        1. Create numpy RNG: rng = np.random.default_rng(seed)
        2. Generate win/loss matrix: outcomes = rng.random((num_paths, num_trades)) < win_rate
        3. For each path (vectorized with numpy):
           - Start with initial_equity
           - Each trade: if win, equity += equity * position_fraction * (avg_win / avg_loss)
                         if loss, equity -= equity * position_fraction
           - Track running peak and max drawdown per path
        4. Compute:
           - max_drawdown_pcts: list of max drawdown % per path
           - final_equities: terminal equity per path
           - ruin_probability: fraction of paths where max_drawdown >= ruin_threshold
           - drawdown_percentiles: np.percentile(max_drawdown_pcts, [50, 75, 90, 95, 99])
           - return_percentiles: np.percentile(returns, [5, 25, 50, 75, 95])
             where returns = (final_equity - initial_equity) / initial_equity * 100

        Use vectorized numpy for performance. The inner loop over trades is sequential
        but the paths dimension is vectorized. For 1000 paths x 200 trades this should
        run in < 2 seconds.

        Log at INFO: "MC simulation: {num_paths} paths x {num_trades} trades, "
                     "ruin_prob={ruin_probability:.3f}, median_dd={p50:.1f}%"
