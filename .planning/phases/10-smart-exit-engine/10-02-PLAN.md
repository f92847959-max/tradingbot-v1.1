# Phase 10 Plan 02: ATR Trailing Stop

<objective>
Implement the smart trailing-stop layer for Phase 10. The trailing stop must activate after a defined profit threshold (+1R by default), move the stop to breakeven when first activated, then trail with ATR distance while never moving the stop in an unfavorable direction.
</objective>

<acceptance_criteria>
- Profit is measured in R multiples from entry to original stop loss.
- No trailing update is emitted before activation_r.
- BUY stops only move up; SELL stops only move down.
- At +1R, stop loss is at least breakeven.
- After price extends, stop trails by ATR * multiplier.
- A stateful manager tracks the last accepted trailing stop per deal_id.
- Unit tests cover BUY, SELL, no-activation, breakeven, ATR-trail, and monotonicity.
</acceptance_criteria>
