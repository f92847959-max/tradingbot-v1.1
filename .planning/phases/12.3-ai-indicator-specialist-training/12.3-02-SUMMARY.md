# Phase 12.3 Plan 02 Summary: Specialist Training and Uplift Comparison

Implemented the isolated specialist training path plus a leak-free Core-vs-Core+Specialist comparison contract.

One-liner: the specialist now trains as its own LightGBM bundle under a separate version root with a local `production.json`, while walk-forward comparison reuses shared windows and per-window train-only scaling so uplift claims stay audit-safe.

## What Changed

- Added `ai_engine/training/specialist_pipeline.py` with `train_specialist_model`, `load_specialist_bundle`, `compare_core_vs_specialist`, and aligned comparison helpers.
- Extended `model_versioning.py` with specialist-only root, version directory, and promotion helpers so specialist artifacts never collide with the core ensemble root.
- Added `serialize_window_spec()` in `walk_forward.py` and used it to emit a stable comparison schema with explicit purge-gap and train-only-scaling metadata.
- Persisted `specialist_lightgbm.pkl`, `specialist_scaler.pkl`, `specialist_version.json`, and `feature_block.json` under `ai_engine/saved_models/specialists/market_structure_liquidity/`.
- Added focused coverage for artifact isolation, feature-block mismatch rejection, comparison schema stability, purge-gap preservation, and misaligned-window failure handling.
