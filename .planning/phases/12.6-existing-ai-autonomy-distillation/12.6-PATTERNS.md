## Reuse Patterns

### 1. Causal teacher snapshot before side effects

Use the 12.4 snapshot discipline as the source pattern.

Implication for 12.6:
- capture decision-time state before broker side effects
- persist observation and realized outcomes separately

### 2. Optional runtime learner with safe fallback

Use `ai_engine/prediction/specialist.py` as the runtime shape.

Implication for 12.6:
- load the autonomy head in-process
- missing artifacts or disabled mode must degrade to current champion behavior

### 3. Governor remains authoritative

Use the current `EnsemblePredictor -> DecisionGovernor` boundary as the preservation point.

Implication for 12.6:
- the head may alter preliminary probabilities/actions
- it must not bypass conflict, threshold, higher-TF, or specialist gates

### 4. Existing governance persistence

Use `GovernanceDecision.evaluation_summary` and `GovernanceDecisionRepository`.

Implication for 12.6:
- rollout metadata belongs in structured JSON on the existing governance record
- no second rollout audit store should be introduced

## Preferred Plan Boundaries

1. Teacher snapshots and label manifest
   Files: capture/export modules, governance logging tests

2. Decision-head training, calibration reuse, and promotion gate
   Files: training pipeline, runtime load contract, walk-forward comparison tests

3. Shadow rollout and guarded runtime selection
   Files: `ensemble.py`, `signal_generator.py`, `trading_loop.py`, governance persistence/tests

## Avoid

- Do not rebuild the dataset only from `signals` and `trades`.
- Do not store giant raw feature blobs in governance logs.
- Do not bypass `StrategyManager`, calendar blocks, or `RiskManager`.
- Do not expand action space or broker semantics in this phase.
