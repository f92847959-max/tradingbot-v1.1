## 2. Create `ai_engine/training/shap_importance.py`

This module provides two public functions:

### Function 1: `compute_shap_importance`

```python
def compute_shap_importance(
    model,
    X_data: np.ndarray,
    feature_names: list[str],
    max_samples: int = 2000,
) -> dict[str, float]:
    """Compute mean absolute SHAP importance for each feature.

    Uses shap.TreeExplainer for exact Shapley values on tree models
    (XGBoost, LightGBM). Handles multi-class output (3 classes:
    SELL/HOLD/BUY) by averaging |SHAP| across samples and classes.

    Args:
        model: Trained XGBClassifier or LGBMClassifier instance
            (the .model attribute from XGBoostModel/LightGBMModel).
        X_data: Feature matrix to explain, shape (n_samples, n_features).
            Should be test/validation data, NOT training data.
        feature_names: List of feature names matching X_data columns.
        max_samples: Maximum samples for SHAP computation. If X_data
            has more rows, a random subsample is used.

    Returns:
        Dict of {feature_name: mean_abs_shap_value}, sorted descending
        by importance.
    """
```

Implementation details:
- Import `shap` at module level (not lazy -- this module exists specifically for SHAP)
- Use `shap.TreeExplainer(model)` -- NOT `shap.Explainer()` (avoid KernelExplainer fallback)
- If `len(X_data) > max_samples`, subsample with `np.random.choice(..., replace=False)` using a fixed seed for reproducibility (`np.random.RandomState(42)`)
- Call `explainer.shap_values(X_shap)` to get SHAP values
- Handle output format: check `isinstance(shap_values, list)` for per-class list, or `ndim == 3` for 3D array, or `ndim == 2` for binary/single output (see RESEARCH.md Pitfall 1)
- Compute `mean_importance = mean(|shap_values|)` across samples and classes
- Return `dict(zip(feature_names, mean_importance.tolist()))` sorted descending by value

### Function 2: `save_feature_importance_chart`

```python
def save_feature_importance_chart(
    shap_importance: dict[str, float],
    output_path: str,
    top_n: int = 20,
) -> str:
    """Save a horizontal bar chart of SHAP feature importance as PNG.

    Args:
        shap_importance: Dict of {feature_name: importance} (from
            compute_shap_importance).
        output_path: File path for the PNG output.
        top_n: Maximum number of features to display (top N by importance).

    Returns:
        The output_path string (for chaining/logging).
    """
```

Implementation details:
- Set `matplotlib.use('Agg')` at MODULE LEVEL (before any pyplot import) to ensure headless rendering
- Import order at top of file: `import matplotlib; matplotlib.use('Agg'); import matplotlib.pyplot as plt`
- Sort features descending, take top N
- Create horizontal bar chart (`ax.barh`) with features on y-axis (least important at top, most important at bottom -- standard SHAP convention with `reversed()`)
- X-axis label: "Mean |SHAP value|"
- Title: "Feature Importance (SHAP)"
- Bar color: `'#1f77b4'` (matplotlib default blue)
- Figure size: `(10, max(6, len(features) * 0.3))`
- `fig.savefig(output_path, dpi=150, bbox_inches='tight')`
- `plt.close(fig)` after save (prevent memory leak)
- Create parent directory if it doesn't exist (`os.makedirs`)
- Return `output_path`

<notes>
- SHAP 0.51.0 has pre-built wheels for Python 3.13 (verified in research). If pip install fails, check platform compatibility.
- The `matplotlib.use('Agg')` call MUST be at module level, before `import matplotlib.pyplot as plt`. If SHAP imports pyplot internally first, this may produce a warning but will still work because SHAP sets Agg by default in its plot utilities.
- Do NOT use `shap.Explainer()` -- it auto-detects model type and can fall back to the slow KernelExplainer. Use `shap.TreeExplainer()` explicitly.
- The `model` parameter accepts the raw sklearn-API model (e.g., `XGBClassifier` instance), not the `XGBoostModel` wrapper. Callers pass `trainer._xgboost.model`.
- For the subsample random seed, use a local `np.random.RandomState(42)` to avoid affecting global state.
- Multi-class SHAP output shape varies by SHAP version -- the isinstance/ndim checks are critical (see RESEARCH.md Pitfall 1).
</notes>
