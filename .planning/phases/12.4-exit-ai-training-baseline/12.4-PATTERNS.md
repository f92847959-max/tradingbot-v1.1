## Reuse Patterns

### 1. Deterministic teacher -> specialist student

Use the Smart Exit Engine modules as the baseline source of truth.

- `dynamic_sl.py` defines tighten semantics
- `dynamic_tp.py` defines TP/TP1 structure context
- `trailing_manager.py` defines monotonic trailing behavior
- `partial_close.py` defines one-shot partial-close contract

Implication for 12.4:
- Exit-AI labels should be derived from, or compared against, these deterministic outputs
- do not invent a parallel exit-rule universe

### 2. Separate specialist artifact root

Use the same artifact isolation principle as Phase 12.3.

Implication for 12.4:
- Exit-AI must have its own version root and pointer file
- it must not overwrite core ensemble or market-structure specialist artifacts

### 3. Walk-forward parity

Use `ai_engine/training/walk_forward.py` as the enforcement pattern.

Implication for 12.4:
- baseline rule path and Exit-AI candidate path must share identical window splits
- any uplift/protection claim without same-window comparison is invalid

### 4. Operator-facing training CLI

Use `scripts/train_models.py` as the CLI/report pattern.

Implication for 12.4:
- a dedicated training entrypoint or mode should emit version/report locations clearly
- artifact metadata should stay JSON-friendly for later audit

## Preferred Plan Boundaries

1. Snapshot builder and action labels
   Files: `ai_engine/training/exit_ai_labels.py`, `tests/test_exit_ai_labels.py`

2. Training pipeline and isolated versioning
   Files: `ai_engine/training/exit_ai_pipeline.py`, `model_versioning` helper(s), `tests/test_exit_ai_model.py`, `tests/test_exit_ai_walk_forward.py`

3. Calibration, promotion gate, and training/report entrypoint
   Files: `ai_engine/training/exit_ai_evaluation.py`, `scripts/train_exit_ai.py`, `tests/test_exit_ai_evaluation.py`

## Avoid

- Do not train directly on post-close trade summaries without timestamped decision snapshots.
- Do not let the model emit actions outside the constrained manifest.
- Do not compare candidate metrics against a different baseline window setup.
- Do not save Exit-AI artifacts into the core `ai_engine/saved_models` root.
