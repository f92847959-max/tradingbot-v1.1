Create `scripts/start_mirofish.py` -- a Python script that handles MiroFish setup (clone, install) and startup (Flask backend). This script automates everything the user would otherwise need to do manually (per CONTEXT.md: "User cannot program -- all setup/configuration must be automated or scripted").

The script must:

1. **Clone MiroFish** (if `mirofish/` does not exist):
   - `git clone https://github.com/666ghj/MiroFish.git mirofish`
   - Run from project root directory

2. **Install dependencies** (if `mirofish/backend/.venv/` does not exist):
   - Check `uv` is available: `uv --version`
   - Run `uv sync` inside `mirofish/backend/` (creates isolated Python 3.11 venv)
   - uv auto-downloads Python 3.11 if not present (camel-oasis requires Python <3.12)

3. **Create MiroFish .env** (if `mirofish/backend/.env` does not exist):
   - Read host project `.env` for `OPENAI_API_KEY` (reuse as `LLM_API_KEY`)
   - Prompt user or read from host `.env` for `ZEP_API_KEY`
   - Write `mirofish/backend/.env` with:
     ```
     LLM_API_KEY={openai_key}
     LLM_BASE_URL=https://api.openai.com/v1
     LLM_MODEL_NAME=gpt-4o-mini
     ZEP_API_KEY={zep_key}
     FLASK_PORT=5001
     ```

4. **Start Flask backend**:
   - Python executable: `mirofish/backend/.venv/Scripts/python.exe` (Windows)
   - Command: `[python_exe] run.py` with cwd=`mirofish/backend/`
   - Use `subprocess.Popen` (non-blocking)
   - Wait up to 15 seconds for health check: `GET http://localhost:5001/health`
   - Print status (started/failed) to stdout

5. **CLI interface**:
   - `python scripts/start_mirofish.py setup` -- clone + install + configure .env
   - `python scripts/start_mirofish.py start` -- start Flask backend
   - `python scripts/start_mirofish.py status` -- health check only
   - `python scripts/start_mirofish.py` (no args) -- setup + start

Use `argparse` for CLI. Use `subprocess.run` for git/uv commands. Use `httpx` (sync) for health check (httpx is already in requirements.txt). Use `pathlib.Path` for all paths. Use `dotenv` for reading host .env. Print all output in German (per D-05 convention) since user is German-speaking.

Do NOT import anything from the trading bot codebase -- this script must be self-contained and runnable independently.
