# Phase 12.7 - Pattern Map

## Purpose

Map planned files to existing analogs so execution follows established GoldBot training patterns instead of creating parallel systems.

## Planned File Patterns

| Planned file | Role | Closest analog | Pattern to reuse |
|--------------|------|----------------|------------------|
| `ai_engine/training/data_coverage.py` | Dataset preflight, row-loss telemetry, manifest writer | `ai_engine/training/data_source.py` | dataclass result objects, explicit custom error, pure helpers with pandas DataFrame inputs |
| `tests/test_training_data_coverage.py` | Unit tests for preflight and manifests | `tests/test_training_data_source.py` | tmp_path CSV/JSON artifacts, direct helper assertions, `pytest.raises` for invalid data |
| `ai_engine/training/causal_training_labels.py` | Label schema wrapper and split manifest | `ai_engine/training/label_generator.py`, `ai_engine/training/decision_distill_dataset.py` | keep existing triple-barrier labels, add manifest metadata, use deterministic DataFrame transforms |
| `tests/test_causal_training_labels.py` | Causal labels and future-mutation tests | `tests/test_decision_snapshot_capture.py`, `tests/test_exit_ai_labels.py` | mutation-safety tests and explicit label/action schema checks |
| `ai_engine/training/promotion_gate.py` | Candidate/champion promotion decision | `ai_engine/training/walk_forward.py`, `ai_engine/training/decision_head_pipeline.py` | compare structured metrics, write explicit decision artifact, do not promote when gates fail |
| `tests/test_training_promotion_gate.py` | Promotion gate tests | `tests/test_walk_forward_calibration.py`, `tests/test_specialist_walk_forward.py` | synthetic metrics, no real model fit required, assert schema and blocking reasons |
| `scripts/train_models.py` | Operator CLI integration | existing `scripts/train_models.py` | preserve legacy flags, add explicit preflight/manifest output, keep dry-run behavior |
| `ai_engine/training/pipeline.py` | Training report integration | existing `TrainingPipeline.run()` | enrich `results` dict and version-dir report without changing live predictor runtime |

## Data Flow

1. Source load:
   - DB/file/broker/synthetic data enters through `scripts/train_models.py`.
   - Real historical backfill should route through `ai_engine/training/data_source.py`.
2. Coverage preflight:
   - `data_coverage.calculate_trainable_span()` checks span before expensive feature/label/model work.
   - `data_coverage.build_row_loss_report()` tracks stage counts.
3. Labeling:
   - Existing `LabelGenerator.generate_labels()` remains the entry-label source.
   - `causal_training_labels.build_causal_label_frame()` adds explicit label columns and manifest metadata.
4. Walk-forward:
   - Existing `WalkForwardValidator` remains the source of window specs, calibration, thresholds, and reports.
5. Promotion:
   - `promotion_gate.evaluate_training_promotion()` consumes champion/candidate summaries and writes allow/block reasons.
   - Runtime loading is not modified in this phase.

## Required Conventions

- Keep tests fast with synthetic DataFrames and metric dictionaries.
- Use `python -m pytest ... -q` commands only, no watch mode.
- Do not introduce network-dependent tests.
- Keep artifact writes under tmp paths in tests.
- Preserve Windows path compatibility and avoid symlinks.
- Keep new helpers importable without broker credentials or live DB connections.

## Non-overlap Boundary

Phase 12.7 may prepare offline/shadow artifacts but must not alter:

- `trading/trading_loop.py`
- live order execution paths
- broker-side trade behavior
- runtime model selection outside training artifact decisions
