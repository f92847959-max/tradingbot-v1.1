    1. In `trading/lifecycle.py`, add `self._paused = False` in `__init__`. Add `self._loop_tasks: list[asyncio.Task] = []` to track running loop tasks. In `start()`, after `self._running = True`, wrap each coroutine in `asyncio.create_task()` and store in `self._loop_tasks` before the `asyncio.gather()`. This enables the API to re-launch loops without calling the full `start()` (which does broker auth, data download, etc). Add a helper method `_restart_loops(self)` that checks if loop tasks are done and creates new ones if needed, sets `_running = True`.

    2. In `trading/trading_loop.py`, at the top of the `while self._running` loop body (before `await self._trading_tick()`), add a check: `if getattr(self, '_paused', False): await asyncio.sleep(self.settings.trading_interval_seconds); continue`. This skips signal generation but keeps the loop alive and responsive to unpause.

    3. In `api/schemas/system.py`, add:
       - `BotControlResponse(BaseModel)` with fields: `success: bool`, `message: str`
       - `BotMetricsResponse(BaseModel)` with fields: `orders_today: int`, `successful_commands_24h: int`, `failed_commands_24h: int`, `api_latency_ms: float`, `db_latency_ms: float`

    4. In `api/routers/system.py`, add three new endpoints and one metrics endpoint:

       `POST /system/start`: If `system._running` is already True and loop tasks are alive, return idempotent success. Otherwise, set `system._running = True`, create new asyncio tasks for `_trading_loop()`, `_position_monitor_loop()`, `_daily_cleanup_loop()`, store them in `system._loop_tasks`. This is a LIGHTWEIGHT restart (no broker re-auth, no data download) -- per research recommendation, only re-enable loops. Return `BotControlResponse`. Anti-pattern to avoid: do NOT create a new TradingSystem instance, do NOT call system.start() (that does full init).

       `POST /system/pause`: Set `system._paused = True`. The trading loop checks `_paused` and skips signal generation when True, but position monitoring continues because `_running` stays True. Return `BotControlResponse`. Idempotent.

       `POST /system/resume`: Set `system._paused = False`. Return `BotControlResponse`. Idempotent.

       `GET /metrics`: Query trade summary count for today using the DB dependency. Return `BotMetricsResponse` with `orders_today` from DB query, `successful_commands_24h=0`, `failed_commands_24h=0`, `api_latency_ms=0.0`, `db_latency_ms=0.0`. These placeholder values match what the frontend expects. Real latency tracking is out of scope for Phase 6. Use the existing `get_db` dependency and `TradeRepository.count_today()`.
