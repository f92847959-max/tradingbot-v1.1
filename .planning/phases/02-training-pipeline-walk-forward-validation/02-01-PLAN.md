## Task 1: Create `ai_engine/training/walk_forward.py` -- WalkForwardValidator class

Create a new file with the following components:

### 1a. WindowSpec dataclass

```python
@dataclass
class WindowSpec:
    window_id: int
    train_start: int   # Always 0 for expanding windows
    train_end: int
    test_start: int    # train_end + purge_gap
    test_end: int

    @property
    def train_size(self) -> int: ...
    @property
    def test_size(self) -> int: ...
```

### 1b. `calculate_walk_forward_windows()` function

Parameters:
- `n_samples: int` -- total number of samples after warmup removal
- `min_train_samples: int = 1500` -- minimum first window training size (reduced from 2000 per research, to accommodate smaller datasets)
- `purge_gap: int = 60` -- gap between train/test (passed from pipeline's dynamic calculation)
- `min_test_samples: int = 200` -- minimum test period size

Algorithm (from research):
- Test size = 20% of total window (= 25% of train size), per user decision
- Expanding/anchored: train_start always 0
- Each window's test_end becomes next window's train_end
- Stop when remaining data is too small for a valid test period
- Returns `List[WindowSpec]`

### 1c. WalkForwardValidator class

```python
class WalkForwardValidator:
    def __init__(self, purge_gap: int = 60, min_train_samples: int = 1500, min_test_samples: int = 200):
        ...

    def calculate_windows(self, n_samples: int) -> List[WindowSpec]:
        """Calculate expanding walk-forward windows. Delegates to calculate_walk_forward_windows()."""

    def run_window(
        self,
        window: WindowSpec,
        X: np.ndarray,
        y: np.ndarray,
        feature_names: List[str],
        trainer: "ModelTrainer",
    ) -> Dict[str, Any]:
        """Execute steps 6-11 for a single walk-forward window.

        This method:
        1. Slices X, y by window indices
        2. Creates a fresh FeatureScaler, fits on window's training data ONLY (TRAIN-02)
        3. Scales train/val/test splits within the window
        4. Trains XGBoost + LightGBM
        5. Runs feature selection (importance-based, using XGBoost)
        6. Re-trains on selected features if any dropped
        7. Tunes trade filter on validation subset
        8. Evaluates ML + trading metrics on test portion
        9. Returns per-window results dict with metrics, trained models, scaler, trade_filters
        """

    def run_all_windows(
        self,
        X: np.ndarray,
        y: np.ndarray,
        feature_names: List[str],
        trainer: "ModelTrainer",
    ) -> Dict[str, Any]:
        """Run walk-forward validation across all windows.

        Returns dict with:
        - "windows": List of per-window results
        - "final_models": Models from last window (production models)
        - "final_scaler": Scaler from last window
        - "final_trade_filters": Trade filters from last window
        - "final_feature_names": Selected features from last window
        """
```

**Critical implementation details for `run_window()`:**

The validation subset split within each window: split the test portion into val (67%) and tune (33%), matching the existing pipeline's approach (line 94-96 of current pipeline.py). But for walk-forward, the "test" portion IS the out-of-sample evaluation. So the split should be:
- Train: `X[window.train_start : window.train_end]`
- Val (for early stopping): last 15% of training data (to mirror current behavior)
- Tune (for trade filter): first part of test period (67%)
- Test (for evaluation): last part of test period (33%)

Actually, re-reading the pipeline more carefully: the current approach uses val for early stopping during training, and tune for trade filter grid search. For walk-forward, a cleaner approach:
- Train: full window training portion
- Val for early stopping: last 15% of training portion (internal split)
- The full test portion is used for evaluation
- Trade filter tuning: use last window's approach -- tune on a held-out portion of the training data or use the test portion from the PREVIOUS window

Simplest correct approach per research: within each window:
- `X_train_full = X[0 : window.train_end]`
- Split X_train_full: 85% train, 15% val (for early stopping + trade filter tuning)
- `X_test = X[window.test_start : window.test_end]` (pure out-of-sample)
- Fit scaler on train portion only
- Train models with train/val
- Tune trade filter on val portion
- Evaluate on test portion

This preserves the existing logic structure while ensuring per-window scaler fitting.

**Reuse existing components** -- do NOT hand-roll:
- Use `trainer._evaluator.evaluate_probabilities()` for ML eval
- Use `trainer._evaluator.evaluate_trading()` for trading eval
- Use `tune_trade_filter()` from `trade_filter.py` for filter tuning
- Use `trainer._xgboost.train()` and `trainer._lightgbm.train()` for model training
- Create fresh `FeatureScaler()` per window (import from `ai_engine.features.feature_scaler`)

## Task 3: Refactor `pipeline.py` to use walk-forward loop

Modify `TrainingPipeline.run()`:

**Keep steps 1-5 OUTSIDE the loop** (unchanged, per research -- indicators are lookback-only, safe on full data):
1. Validate data + add 6-month check (TRAIN-07)
2. Compute features (on full dataset -- NO leakage for lookback indicators)
3. Generate labels
4. Remove warmup period
5. Separate features and labels

**Replace steps 6-12 with walk-forward**:
6. Create `WalkForwardValidator` with dynamic purge gap
7. Calculate windows via `validator.calculate_windows(len(X))`
8. Log window structure
9. Call `validator.run_all_windows(X, y, feature_names, self._t)`
10. Extract final models, scaler, trade filters, feature names from last window
11. Save models (delegate to Plan 02-02's versioned saving -- for now, save to existing paths as baseline)
12. Save metadata (delegate to Plan 02-02's version.json -- for now, extend existing metadata with walk-forward results)

**The `run()` method signature stays the same** -- `ModelTrainer.train_all()` continues to work identically from the caller's perspective.

Update step numbering in log messages to reflect new flow (fewer explicit steps since walk-forward handles 6-11 internally).

**Important:** The `run()` method must return the same `results` dict structure for backward compatibility, plus additional walk-forward data:
- `results["walk_forward_windows"]` -- list of per-window result dicts
- `results["n_windows"]` -- number of windows
- All existing keys (`metadata`, `xgboost_train`, etc.) should still be present, populated from the LAST window's results

<notes>
- The WalkForwardValidator.run_window() must create a FRESH FeatureScaler per window. Do NOT reuse trainer._scaler across windows -- that would leak scaling statistics. Import FeatureScaler and instantiate a new one each time.
- The existing trainer._xgboost and trainer._lightgbm models will be overwritten each window (that's fine -- we only keep the last window's models for production).
- The dynamic purge gap calculation from the current pipeline (line 88: `min(max_label_horizon, max(8, len(X) // 20))`) should be computed ONCE before the loop and passed to WalkForwardValidator.
- For feature selection: run it per-window using XGBoost importance. This means features may differ across windows. The FINAL window's selected features are the production features.
- Window type is EXPANDING (anchored) per research recommendation -- train_start is always 0.
- Test period = 20% of total window size per user decision. Mathematically: test_size = train_size * 0.25 (since 20% of total = 25% of train).
- min_train_samples set to 1500 (not 2000) per research open question about limited Capital.com data.
</notes>
