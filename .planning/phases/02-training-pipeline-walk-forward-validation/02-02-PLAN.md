## Task 1: Create `ai_engine/training/model_versioning.py`

### 1a. `create_version_dir(base_dir: str) -> str`

Creates a new versioned directory:
- Format: `v{NNN}_{YYYYMMDD}_{HHMMSS}` (e.g., `v001_20260306_143022`)
- Scans existing `v*` directories to determine next number
- Creates the directory
- Returns the full path

### 1b. `write_version_json(version_dir: str, version_data: dict) -> str`

Writes `version.json` to the version directory.

The version.json schema (extends existing model_metadata.json):
```python
{
    # From existing metadata (MUST keep all existing fields for backward compat)
    "training_date": str,           # ISO format
    "timeframe": str,
    "n_samples_total": int,
    "n_features_original": int,
    "n_features_selected": int,
    "feature_names": List[str],
    "label_params": dict,           # tp_pips, sl_pips, max_candles, pip_size, spread_pips, slippage_pips
    "training_duration_seconds": float,

    # NEW: Version metadata
    "version": str,                 # e.g., "v001"
    "version_dir": str,             # directory name

    # NEW: Data range info
    "data_range": {
        "n_candles": int,
        "months_of_data": float,    # computed from index if DatetimeIndex available
    },

    # NEW: Walk-forward results (TRAIN-05: version metadata includes metrics)
    "walk_forward": {
        "n_windows": int,
        "window_type": "expanding",
        "purge_gap_candles": int,
        "windows": [
            {
                "window_id": int,
                "train_samples": int,
                "test_samples": int,
                "metrics": {
                    "xgboost": {
                        "accuracy": float,
                        "f1": float,
                        "win_rate": float,
                        "profit_factor": float,
                        "expectancy": float,
                        "n_trades": int,
                    },
                    "lightgbm": { ... same structure ... },
                },
            },
        ],
    },

    # NEW: Aggregate metrics across all windows
    "aggregate_metrics": {
        "xgboost": { "win_rate": float, "profit_factor": float, "expectancy": float, "n_trades": int },
        "lightgbm": { ... },
    },

    # Existing per-model flat metrics (kept for backward compat)
    "xgboost_accuracy": float,
    "xgboost_f1": float,
    # ... etc
}
```

### 1c. `update_production_pointer(base_dir: str, version_dir: str) -> None`

- Writes `production.json` to base_dir with: `{"version_dir": "v001_...", "updated": "ISO", "path": "full/path"}`
- Copies `xgboost_gold.pkl`, `lightgbm_gold.pkl`, `feature_scaler.pkl` from version_dir to base_dir (backward compatibility -- existing code loads from base_dir)
- Also copies `version.json` as `model_metadata.json` to base_dir (extending the existing file, not replacing the concept)

### 1d. `cleanup_old_versions(base_dir: str, keep: int = 5) -> List[str]`

- Lists all `v*` directories, sorts by version number
- Deletes directories beyond the `keep` most recent
- Returns list of deleted directory names
- Logs what was deleted

<notes>
- The version.json schema EXTENDS the existing model_metadata.json. All existing flat keys (xgboost_accuracy, lightgbm_win_rate, etc.) must still be present at the top level for any code that reads model_metadata.json directly.
- `update_production_pointer` copies files to base_dir AND writes production.json. This means `ai_engine/saved_models/xgboost_gold.pkl` always points to the latest production model -- existing live trading code works without modification.
- Use `shutil.copy2` for file copying (preserves metadata).
- Use `shutil.rmtree` for cleanup of old version directories.
- Windows compatibility: use `production.json` pointer file (NOT symlinks -- symlinks require admin on Windows).
- The `version.json` `walk_forward.windows[].metrics` should include both models (xgboost, lightgbm) with their per-window results. If a model failed to train in a window, its metrics should be an empty dict or contain `{"error": "..."}`.
- TRAIN-02 compliance: the scaler saved in the version directory is from the LAST window, fitted on that window's training data only. This is verified by the walk-forward engine (Plan 02-01) creating a fresh scaler per window.
</notes>
