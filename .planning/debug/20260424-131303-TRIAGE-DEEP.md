## Revised Findings (sorted by severity)

### [HIGH] `math >=0.1.0` in `requirements.txt:27` and `pyproject.toml` dependencies
- Evidence: `requirements.txt:27` reads literally `math >=0.1.0`. `math` is a Python **stdlib builtin module** (`math.__name__ in sys.builtin_module_names == True`), not a PyPI package.
- Impact: `pip install -r requirements.txt` will resolve and install the PyPI package named `math` — whoever owns that name on PyPI gets code execution in anyone's venv who installs from this file. Builtin shadowing means stdlib still wins at import time, so **runtime behavior looks normal** — but a supply-chain vector is open.
- 1285 `import math` / `from math import` statements across the codebase rely on the builtin; they don't need a PyPI dep.
- Next: remove line 27 of `requirements.txt` and the equivalent entry in `pyproject.toml` `dependencies`. `pip uninstall math` in `.venv` to purge whatever got installed.
